# 自动化测试候选用例标准（automation-testcases-standard）

## 适合自动化的特征

满足以下特征的用例适合纳入自动化候选：
- **稳定可重复**：相同输入每次运行结果一致，不依赖随机因素
- **高执行频率**：频繁回归的核心功能，投入产出比高
- **断言明确**：结果可精确判断 pass/fail，不依赖主观判断
- **接口或可脚本化**：通过 API/CLI / 脚本可驱动，无需人工操作 UI
- **依赖可构造**：前置数据可通过脚本/SDK 构造，不依赖人工准备

## 不适合或低优先的特征

以下情况不建议自动化或优先级别低：
- 强主观/探索性（需人工判断体验、UI 细节）
- 一次性/低频（几乎不回归，一次性验证）
- 环境或数据难以自动化（依赖特殊硬件、手工数据）
- 变更频繁（页面结构、流程经常变化，维护成本高）

## 输出要求

在用例的「类型」或「备注」字段标注：
- `自动化候选` — 推荐优先实现自动化
- `建议自动化` — 可自动化但优先级一般
- `低优先级` / `不建议自动化` — 原因已在备注中说明

可选补充：
- 建议实现方式（API 自动化 / UI 自动化 / 性能自动化）
- 关键断言要点
- 数据与环境要求
