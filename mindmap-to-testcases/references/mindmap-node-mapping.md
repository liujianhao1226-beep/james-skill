# 思维导图节点到用例字段的映射规则

## 先路径抽取，再字段映射

不要直接按单个节点生成用例。先从根节点走到叶子节点，形成完整路径，再判断每个节点在该路径中的角色。

## 注释节点处理

以下前缀开头的节点是注释/备注，不是测试路径的一部分：

- `注：`、`注:`、`备注：`、`备注:`
- `说明：`、`说明:`、`参考：`、`参考:`
- `提示：`、`提示:`、`NOTE:`、`TODO:`、`FIXME:`

处理方式：
- 脚本解析时会自动标记为 `is_annotation: true`
- 注释内容可以作为前置条件的补充说明或用例备注，但不应生成独立用例
- 如果注释包含测试约束（如"考虑App对离线设备有重新发起连接的入口"），应将其融入相关用例的前置条件或步骤中

## 常见节点类别

### 1. 模块/功能域
通常是目录型上层节点，例如：
- 组网管理、HAC组网、AC组网
- 功能、配网、远程控制、语音识别
- 屏幕交互、灯控、场景模式
- 性能、稳定性、兼容性、产测、嵌入式测试

这类节点通常进入 `所属模块`。

### 2. 场景标签
通常是概括测试意图的中层节点，例如：
- HAC DHCP开局、HAC PPPoE开局
- 页面无线组网、按键无线组网
- 设备触发配网、手机网络连接
- 语音控制灯光、屏幕滑动切换、OTA升级流程
- 多端同时控制、定时任务执行

这类节点通常进入标题中的 `【场景标签】`。

### 3. 前置条件
常见模式：
- 上网方式：dhcp / pppoe / 静态 ip
- 设备拓扑：并联组网 / 串联组网 / 交换机拓展 / 无线回程
- 数据状态：已复位 / 未初始化 / 已绑定 / 已组网
- 依赖条件：已有账号权限 / 已有 HAC / 已有 AP / 已存在历史配置
- 机型组合：HAC + GTW06B + GTW05B + GTW04A
- 网络环境：手机4G网络 / 手机5G网络 / WiFi弱网 / 局域网 / 外网
- 设备状态：在线 / 离线 / 产测模式 / 已配对 / 未配对
- 绑定关系：同一账号 / 不同账号 / 不同家庭

这些一般进入 `前置条件`。但如果节点表达的是状态切换后的动作，例如"完成配置向导后""重启后"，则要放入 `步骤` 或 `状态迁移`，而不是只放前置条件。

### 4. 操作步骤
常见动词：
- 完成配置向导、接入 AP、页面点击添加、按键触发
- 重启、复位、断电重上、插拔网线、登录页面
- 发起配网、扫码绑定、语音唤醒、下发指令
- 滑动切换、旋钮调节、长按触发、连续点击
- 升级固件、回退版本、恢复出厂设置
- 开关灯、调亮度、切换色温、设置定时
- 杀后台、切后台、关闭蓝牙、断开WiFi

这类节点通常进入 `步骤`。

### 5. 预期结果
常见结果词：
- 成功 / 失败 / 不支持 / 不可用
- 可上线 / 不可上线 / 可登录 / 不可登录
- 可上网 / 不可上网 / 可控制 / 不可控制
- 弹出提示 / 不弹提示 / 显示正常 / 显示异常
- 获取到地址 / 无法获取地址
- 配网成功 / 配网失败 / 配网超时
- 语音响应正确 / 语音无响应 / 误唤醒
- 状态同步 / 状态不一致 / 恢复正常
- 亮度/色温/模式生效 / 不生效

这类节点通常进入 `预期`。如果叶子只写了"成功"，则必须结合上一层动作，把结果补成可观察句子。

### 6. IoT 特有维度节点

以下节点类型在智能硬件/IoT 测试中常见，需要特殊处理：

#### 通信协议/配网方式
- 蓝牙配网、WiFi配网、Zigbee配网、Matter配网
- 2.4G WiFi / 5G WiFi / 双频合一
- 这类节点通常作为 `前置条件` 或 `场景标签`

#### 控制端/交互方式
- App控制、语音控制、屏幕触控、物理按键、旋钮
- 本地控制、远程控制、自动化/定时
- 不同控制端通常应拆为独立用例

#### 传感器/感应
- 接近感应、光感、人体感应、温湿度
- 感应距离（远/中/近）、灵敏度等级
- 这类节点通常作为 `前置条件` 或独立测试维度

#### 固件/OTA
- OTA升级、回退版本、升级中断、升级后验证
- 这类节点通常生成独立的状态迁移用例

#### 产测/工厂模式
- 产测模式、老化测试、功能隔离、校准
- 这类节点通常属于独立模块，不要和正常功能测试混在一起

## 高价值拆例信号

出现下列词语时，通常应拆为独立用例，而不是揉成一条：
- 重启后、复位后、一起上电、重新接入
- 页面方式 / 按键方式 / 语音方式
- 并联 / 串联
- dhcp / pppoe / 静态 ip
- 已绑定 / 未绑定
- 成功 / 失败 / 不支持
- 可上线 / 不可上线
- App控制 / 本地控制 / 语音控制（不同控制端）
- 在线状态 / 离线状态
- 正常网络 / 弱网环境
- 左右滑动 / 上下滑动 / 旋钮操作（不同交互手势）

## 不要机械展开的内容

默认不要大量拆例：
- 仅设备型号不同但验证目标相同
- 仅版本号或 build 号不同
- 仅表述细节不同但测试意图一致
- 仅网络类型不同（4G/5G）但配网流程完全一致——可合并为 1 条，网络类型写入前置条件

只有当"兼容性本身就是测试点"时，才对型号或版本做单独拆分。

## 标题拼装顺序

推荐优先级：
1. 场景标签
2. 核心动作/状态迁移
3. 关键条件（如 DHCP / PPPoE / 并联 / 串联 / 蓝牙 / WiFi）
4. 结果

示例：

`【配网】手机连接5GWiFi，蓝牙配网-配网成功`
`【语音控制】语音指令开灯-灯光正常开启`
`【屏幕交互】左右横滑切换页面-循环切换正常`
`【OTA升级】升级过程断电-重新上电后可继续升级或回退`

## 生成时的判断顺序

1. 这条路径是否包含动作？
2. 这条路径是否包含可观察结果？
3. 哪些节点是静态条件，哪些节点是动态动作？
4. 是否存在状态迁移信号？
5. 是否存在不同控制端/交互方式需要拆例？
6. 是否有注释节点需要融入前置条件或步骤？
7. 是否需要额外补异常/边界/场景？
