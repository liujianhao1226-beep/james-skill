# 输入归一化规则

先把不同来源统一成“根到叶路径”，再生成测试用例。不要直接按单个节点输出用例。

## 支持的输入类型

### 1. 思维导图图片（PNG/JPG/JPEG）
- 使用视觉能力先提取主干层级
- 输出成层级清单或路径列表
- 只保留与测试意图有关的节点
- 看不清时优先保主干，不要硬猜型号或版本

### 2. XMind 原文件（.xmind）
- 优先运行 `scripts/extract_mindmap_paths.py`
- 支持常见 `content.json`、`workbook.json`、`content.xml` 结构
- 默认按 sheet 分组，抽取根到叶路径

### 3. Markdown / 层级文本（.md/.markdown/.txt）
- 支持标题 + 列表混合形式
- 支持纯列表和缩进树
- 支持 `-`、`*`、`+`、数字列表，以及简单的缩进文本

### 4. CSV 路径导出（.csv）
默认支持两类：
- 多列层级：`level1, level2, level3...`
- 单列路径：`A -> B -> C` / `A / B / C`

如果 CSV 已经像真实测试用例表，表头包含 `所属模块/用例标题/前置条件/步骤/预期` 等设计列，则将它视为**参考样例**而不是待转换树源。

## 归一化单位

归一化输出的最小单位是：

```json
{
  "path": ["组网管理", "HAC组网", "HAC DHCP开局", "完成配置向导", "再接入AP", "有线组网成功"],
  "path_text": "组网管理 -> HAC组网 -> HAC DHCP开局 -> 完成配置向导 -> 再接入AP -> 有线组网成功"
}
```

## 提取时保留什么

必须保留：
- 模块路径
- 场景标签
- 上网方式、拓扑、状态、角色等条件
- 动作节点
- 结果节点
- 数量、范围、边界、限制词

默认不当作主路径主体的内容：
- 版本执行结果列
- 纯 build 号
- 单纯为了记录测试轮次而追加的列

## CSV 识别启发式

### 优先识别为路径 CSV 的情况
- 表头含 `path/路径/层级/level/l1/l2/...`
- 每行前几列就是一条分层路径
- 单列中出现 `->`、`>`、`/` 这类路径分隔符

### 优先识别为参考用例表的情况
- 表头包含 `所属模块` 和 `用例标题`
- 同时包含 `前置条件/步骤/预期`
- 后面还有大量版本或执行列

这种情况下，不要把现成用例表再次拆回树；只用来对齐命名风格、字段顺序或做差异比对。

## 图片场景的 best-effort 原则

对截图做转写时：
- 先抽主要分支，再补细分节点
- 优先识别动作和结果
- 型号枚举、版本矩阵、右侧长备注可以后处理
- 无法确认时标记“待确认”，不要伪造确定性
