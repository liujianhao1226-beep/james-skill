# 输出字段与文件约束

本 skill 默认只生成实际测试用例表的前 8 列：

1. `所属模块`
2. `用例标题`
3. `前置条件`
4. `步骤`
5. `预期`
6. `优先级`
7. `用例类型`
8. `适用阶段`

## 固定规则

- 默认不要追加版本执行列
- 默认不要追加备注列、负责人列、结果列
- 如果用户明确要 `.csv/.xlsx`，文件中也只输出这 8 列
- 字段顺序不可变

## 字段填写规则

### 所属模块
- 使用稳定模块路径，例如：`/组网管理/HAC组网`
- 不把机型、版本、执行结果列塞进模块路径

### 用例标题
推荐格式：`【场景标签】谁在什么条件下做什么-预期什么`

### 前置条件
- 建议逐条列出
- 写环境、拓扑、设备、数据状态、权限依赖
- 不写执行动作

### 步骤
- 逐步编号
- 每步一个主动作
- 必须能在测试环境复现

### 预期
- 逐条编号
- 尽量写可观察结果：页面状态、上线数量、联网状态、提示文案

### 优先级
默认使用 `1/2/3`：
- `1`：核心主流程、核心兼容场景
- `2`：重要分支、状态迁移、典型异常
- `3`：长尾异常、补充边界

### 用例类型
常用值：`功能测试`、`异常测试`、`边界测试`、`场景测试`

### 适用阶段
默认 `版本验证阶段`

## JSON 中间格式

导出脚本支持两套字段名：

### 中文字段
- `所属模块`
- `用例标题`
- `前置条件`
- `步骤`
- `预期`
- `优先级`
- `用例类型`
- `适用阶段`

### 英文字段别名
- `module`
- `title`
- `preconditions`
- `steps`
- `expected`
- `priority`
- `case_type`
- `stage`

## XLSX 导出约定

输出 `.xlsx` 时，默认：
- sheet 名称：`testcases`
- 冻结首行
- 开启筛选
- 单元格自动换行
- 保持 8 列顺序不变

## CSV 导出约定

输出 `.csv` 时：
- 使用 UTF-8 with BOM，便于 Excel 直接打开中文
- 多行字段用换行符保存
- 保持 8 列顺序不变
