# Manual Baseline Alignment

当用户同时提供思维导图和一份人工编写的测试用例表时，不要默认“导图叶子节点 = 最终交付用例”。先把人工版当作粒度基线，再决定最终输出应该保留多少细节。

## 先判断人工版的粒度

优先抽样查看 8 到 20 条人工用例，判断它更接近哪一种：

- `叶子直出`：一条叶子路径基本对应一条用例。
- `场景聚合`：一个父节点下的多个叶子会被合并成 1 条完整场景。
- `检查清单`：多个默认值、显示项、状态项会合并到同一条检查型用例里。

如果人工版明显偏向 `场景聚合` 或 `检查清单`，最终交付时应优先对齐人工版粒度；原始叶子展开可以作为中间稿保留，但不要直接当成最终表格。

## 识别导图和人工用例之间的关系类型

### 1. 直接映射

人工用例与导图中的一条叶子路径几乎等价，可以直接保留为 1 条用例。

示例：

- 导图：`产测 -> 产测状态：不能配网及配对`
- 人工版：`【产测状态】不能配网及配对`

### 2. 同一场景下的兄弟叶子聚合

如果多个叶子属于同一个场景父节点，且这些叶子只是该场景下的结果面、能力面或检查面，应合并成 1 条场景用例。

示例：

- 导图：`上电初始化 -> 不绑定米家APP，选离线模式 -> 设备进入离线模式，并自动跳转到首页`
- 导图：`上电初始化 -> 不绑定米家APP，选离线模式 -> 可以控制2路灯控1路调光`
- 导图：`上电初始化 -> 不绑定米家APP，选离线模式 -> 修改显示设置、屏保、本地功能等`
- 人工版合并为：
  `【出厂默认状态】未绑定米家APP，离线模式`

### 3. 生命周期连续叶子聚合

如果多个叶子描述的是同一次用户旅程里的连续阶段，应合并成 1 条端到端用例，而不是拆成多条碎片。

示例：

- 导图：`上电初始化 -> 启动完成 -> 启动时间≤10s`
- 导图：`上电初始化 -> 启动完成 -> 启动完成 -> 显示二维码图片...同时进入自配网状态`
- 人工版合并为：
  `【出厂默认状态】整机启动时间及启动完成屏幕显示`

### 4. 默认值检查清单聚合

如果父节点本身就是 `默认状态`、`检查默认状态`、`出厂配置` 之类的检查场景，子叶子通常不应一叶一例，而应聚合成 1 条清单式校验用例。

示例：

- 导图：`默认亮度80%`
- 导图：`默认首页灯控页`
- 导图：`默认屏保1`
- 导图：`默认待机30s`
- 导图：`默认接近感应=远`
- 导图：`默认勿扰=22:00-07:00`
- 人工版合并为：
  `【出厂默认状态】扫码绑定米家APP，检查默认状态`

### 5. 触发方式或风险域不同，保留拆分

不要为了贴近人工版而过度合并。只要验证目标明显不同，就应该保留独立用例。

典型需要保留拆分的信号：

- 触发方式不同：左右滑动、下拉呼出、点击、旋钮、语音、App、物理按键。
- 风险域不同：正常交互、返回逻辑、压力测试、稳定性、烧屏、长时间运行。
- 结果类型不同：成功、失败、恢复、超时、取消/确认分支。

示例：

- `左右横滑，循环切换`
- `自定义页面显示顺序`
- `主页面向下滑动可呼出设置页`
- `点击效果提示`
- `快速连续滑动不会导致页面错乱或崩溃`
- `多点触控/斜滑稳定性`
- `烧屏测试`

这类场景即使都属于 `屏幕交互`，通常也应保留为多条独立用例。

## 用人工版反推模块命名

不要默认把导图路径原样写成最终 `所属模块`。

如果人工版已经把多个导图分支收敛到一个更稳定、对测试团队更友好的模块名，优先沿用人工版风格。

示例：

- 自动展开可能写成：
  - `/上电初始化/不绑定米家APP，选离线模式`
  - `/上电初始化/扫码添加米家APP`
  - `/上电初始化/出厂默认状态`
- 人工版可能统一写成：
  - `上电初始化/出厂默认状态`

当人工版明显采用“业务阶段模块”而不是“导图原路径模块”时，最终交付优先使用业务阶段模块。

## 用人工版校准优先级、类型和阶段

不要只靠叶子关键词做类型判断。

常见误判：

- `不能配网及配对` 在 `产测功能隔离` 语境下，可能是功能限制校验，不一定要归为异常测试。
- `离线模式` 在首启流程里可能是正常主流程分支，不一定要归为异常测试。

优先按下面顺序判断：

1. 先看父场景是不是主流程、能力校验、限制校验、压力校验。
2. 再看叶子文本是否只是结果描述。
3. 如果人工版已有明确类型、优先级、阶段风格，优先对齐人工版。

## 推荐输出策略

当同时存在导图和人工基线时，优先采用两层产物思路：

1. `raw expansion`：完整叶子展开稿，用于追溯覆盖。
2. `aligned final table`：按人工基线粒度聚合后的最终交付表。

如果用户没有明确要求保留两层产物，优先交付 `aligned final table`。

## 快速检查清单

最终导出前至少检查下面 6 件事：

- 是否仍有大量“一叶一例”的碎片，而人工版明显在做场景聚合。
- 是否把多个默认值分别拆成多条，而人工版是 1 条清单用例。
- 是否把同一次首启/绑定/恢复旅程拆成了多条中间态碎片。
- 是否把导图原路径机械当成 `所属模块`，导致模块名不自然。
- 是否仅凭 `不能/失败/离线` 等词把用例误判成异常测试。
- 是否已经去掉“导图映射”“按导图路径进入”这类生成痕迹。
