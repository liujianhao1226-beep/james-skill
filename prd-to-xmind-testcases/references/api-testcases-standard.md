# 接口测试用例标准（api-testcases-standard）

## 必须覆盖

### 请求与参数
- 正常请求（必填参数齐全、合法取值）
- 必填缺失（逐个缺失必填参数）
- 类型/格式错误（参数类型不符、格式不匹配）
- 边界值（最大值、最小值、超界值）
- 可选参数不传 / 传空 / 传有效值

### 响应与错误码
- 成功响应结构验证（字段完整性、取值合理性）
- 文档中每个错误码至少 1 条用例
- 非法请求不返回 200（返回对应错误码）

### 鉴权与权限
- 未带鉴权 header/token
- 鉴权无效或已过期
- 越权访问（低权限账号访问高权限资源 / 跨用户数据访问）

### 幂等与并发（若文档有说明）
- 重复提交（同一请求多次提交，验证幂等性）
- 超并发 / 限流（超过 QPS 上限的请求处理）

## 表述要求

每条用例写清：
- 接口路径 + HTTP 方法
- 请求关键取值（可用占位符如 `{{userId}}`）
- 预期 HTTP 状态码
- 预期响应体或错误码

数据依赖在前置条件或数据要求中说明。
