# 性能测试用例标准（performance-testcases-standard）

## 必须覆盖

### 指标与基线
- 响应时间（RT）：平均、P50、P90、P99
- 吞吐量 / QPS：系统可承受的每秒请求数
- 资源与容量：CPU、内存、IO（若文档有给出）

### 场景类型
- 基准 / 基线（单用户或低并发下的性能表现）
- 稳态负载（持续正常压力）
- 峰值 / 尖峰（短时突发流量）
- 长时间稳定性（持续压力下的表现，关注内存泄漏等）

### 瓶颈与退化
- 阶梯加压（逐步增加并发，找出瓶颈点）
- 降级与限流（触发降级/熔断的条件与表现，若文档有）

## 表述要求

- 场景名称需体现「场景类型 + 目标指标」，如「峰值负载-P99 RT < 200ms」
- 写清负载定义：并发数 / QPS、时长、预热时间
- 写清环境与数据量级（数据量决定性能表现）
- 写清通过标准：如「P99 RT < 200ms，成功率 >= 99.9%」
- 可选观察要点：CPU、内存、错误日志等
