# Daily Digest — 每日对话精华日记

将一天的对话记录提炼成结构化、可回顾的精华日记。

## 核心逻辑

读取当日的 `memory/YYYY-MM-DD.md` 文件（如不存在则读取昨日），从中提取：
- 关键决策和结论
- 完成的重要事项
- 遇到的问题及解决方案
- 未完成有待跟进的事项
- 有价值的技术积累或想法

然后按结构输出，不做流水账。

## 输出格式

```markdown
# 📅 日记 · YYYY年MM月DD日 · 周X

## ✅ 今日完成
<!-- 列出完成的事项，每条1-3句话，带结果 -->

## 💡 关键决策
<!-- 列出今日做出的重要决定，简要写原因 -->

## 🔧 问题与解决
<!-- 遇到的问题 + 解决方案，格式：问题 → 方案 -->

## ⏳ 待跟进
<!-- 未完成的事项，带优先级标注 -->
<!-- P0 = 必须处理 / P1 = 尽快处理 / P2 = 有空再处理 -->

## 📝 今日收获
<!-- 学到的新东西、优化思路、技术笔记等 -->
```

## 工作流

### 触发后

1. **确定日期**：取当天（`date +%Y-%m-%d`），找不到则取昨日
2. **读取 memory 文件**：`~/.openclaw/agents/feishu-bot/workspace/memory/YYYY-MM-DD.md`
3. **提炼生成**：按上方格式输出精华日记
4. **保存**：
   - 默认保存到 `~/.openclaw/agents/feishu-bot/workspace/memory/diary/YYYY-MM-DD.md`
   - 若用户指定了路径，按用户指定的来
5. **打印保存路径**，告知用户

### 文件不存在时

若 memory 文件不存在，告知用户「今天还没有对话记录，无法生成日记」。

## 生成原则

- **不说废话**：不写「今天做了很多事情」「学到了很多」这类空洞表达
- **具体到结果**：每条记录都指向具体事项和结果
- **区分已完成和未完成**：已完成写结果，未完成的写现状和卡点
- **保留关键上下文**：重要的讨论、决策原因要写进去，方便以后回溯
